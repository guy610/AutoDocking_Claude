dev = ["pytest>=7.0", "pytest-qt>=4.2"]

Tests to write:

1. test_worker_signals_exist():
   - Verify WorkerSignals has: started, progress, finished, result, error signals
   - Just attribute existence check

2. test_worker_executes_function():
   - Create Worker with simple function that returns "success"
   - Use QSignalSpy or manual signal tracking to capture result
   - Run worker synchronously (worker.run() directly, not via threadpool)
   - Assert result signal emitted with "success"
   - Assert finished signal emitted

3. test_worker_emits_progress():
   - Create Worker with function that calls progress_callback(50, 100, "halfway")
   - Capture progress signal emissions
   - Assert progress(50, 100, "halfway") was emitted

4. test_worker_handles_exception():
   - Create Worker with function that raises ValueError("test error")
   - Capture error signal
   - Assert error signal emitted with tuple containing ValueError
   - Assert result signal NOT emitted

5. test_worker_cancellation():
   - Create Worker with function that:
     * Loops 10 times, checking is_cancelled() each iteration
     * Returns "completed" if not cancelled
   - Call worker.cancel() before running
   - Run worker
   - Assert result is NOT "completed" (either None or cancelled early)

6. test_worker_passes_kwargs():
   - Create Worker with function accepting **kwargs
   - Pass custom_arg="custom_value" to Worker
   - Assert function receives custom_arg

Use pytest fixtures for QApplication if needed (pytest-qt provides qtbot fixture).

Note: For synchronous testing, call worker.run() directly. For async testing with QThreadPool, use qtbot.waitSignal().
