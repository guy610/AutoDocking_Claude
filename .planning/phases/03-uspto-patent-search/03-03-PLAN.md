<task type="auto">
  <name>Task 1: Integrate search into MainWindow</name>
  <files>src/fto_agent/main_window.py</files>
  <action>
Update MainWindow to wire up the complete USPTO search flow.

**Updates to MainWindow class:**

1. **Add imports:**
   ```python
   from fto_agent.workers import create_uspto_search_worker
   from fto_agent.widgets import ResultsPanel
   from fto_agent.services import PatentSearchResponse, USPTOSearchError
   import os
   ```

2. **Update _setup_central_widget:**
   - Create horizontal splitter (QSplitter with Qt.Horizontal)
   - Left side: InputPanel (existing)
   - Right side: ResultsPanel (new)
   - Set initial splitter sizes: [400, 400] (50/50 split)
   - Store ResultsPanel in self._results_panel

3. **Update _start_fto_search:**
   - Remove the placeholder message
   - Get data from InputPanel.get_data()
   - Check if "US" in data["countries"]; if not, show status message "USPTO search requires US country selection" and return
   - Get API key from os.environ.get("PATENTSVIEW_API_KEY")
   - If no API key, show status message "PATENTSVIEW_API_KEY environment variable not set" and return
   - Disable InputPanel
   - Set ResultsPanel to loading state
   - Create worker: self._current_worker = create_uspto_search_worker(data, api_key)
   - Connect signals:
     - worker.signals.progress -> self._on_progress
     - worker.signals.result -> self._on_search_result (new method)
     - worker.signals.error -> self._on_search_error (new method)
     - worker.signals.finished -> self._on_finished
   - Start progress manager: self._progress_manager.start(worker)
   - Start worker: self._thread_pool.start(self._current_worker)

4. **Add _on_search_result(self, result: PatentSearchResponse):**
   - Call self._results_panel.set_results(result)
   - Show status message: f"Found {result.total_hits} patents"

5. **Add _on_search_error(self, error_info: tuple):**
   - Extract error message from error_info[1] (the exception value)
   - If error_info[1] is USPTOSearchError: use its message
   - Otherwise: use str(error_info[1])
   - Call self._results_panel.set_error(message)
   - Show status message: f"Search failed: {message}"

6. **Update _on_finished:**
   - Call self._results_panel.set_loading(False)
   - Keep existing code (re-enable InputPanel, stop progress, clear worker ref)

The UI should now support the complete flow:
- User fills form and clicks Submit
- Progress bar appears, cancel button available
- Results appear in right panel
- Errors shown in results panel and status bar
