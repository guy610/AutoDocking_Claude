                    STEPHEN DOCKING v0.7.0 - INSTALLATION & USAGE GUIDE
              Automated Peptide Optimization Pipeline with AutoDock Vina
================================================================================


WHAT IS STEPHEN DOCKING?
========================

Stephen Docking is a computational tool that finds the best peptide binder for
a protein target of interest. It is designed for bench scientists who want to
optimize peptide-protein interactions without needing to write code.

How it works:
  - You provide a protein structure (PDB file) and a starting peptide sequence.
  - The tool uses AutoDock Vina, a widely-used molecular docking engine, to
    predict how strongly each peptide variant binds to your protein.
  - It automatically generates and tests thousands of mutations, including:
      * All 20 natural amino acids at each position
      * Unnatural amino acids (UAAs) you define via SMILES
      * Backbone modifications (beta-amino acids, N-methyl variants)
      * N-terminal modifications (methylation, acylation, custom SMILES)
      * C-terminal cap scanning (acid vs amide forms)
      * Residue deletions to find minimal binding motifs
  - Results are ranked by predicted binding energy (kcal/mol).

The pipeline runs in three Vina stages, with an optional 4-phase hierarchical
screening mode that adds GNINA and RxDock for consensus ranking:

  Stage 1 - SIDECHAIN OPTIMIZATION
    Tests all selected natural and unnatural amino acid substitutions at each
    position of your peptide. Keeps the top performers from each round.

  Stage 2 - BACKBONE MODIFICATION
    Takes the best peptides from Stage 1 and tests backbone variants such as
    beta-amino acids and N-methylated residues. These modifications can improve
    protease resistance and bioavailability.

  Stage 3 - MINIMIZE (Residue Deletion)
    Systematically removes residues from the optimized peptide to find the
    shortest sequence that retains strong binding. Shorter peptides are cheaper
    to synthesize and often have better cell permeability.

  HIERARCHICAL MODE (optional, requires GNINA and/or RxDock):
    After the three Vina stages, the top 20 candidates are carried through:

    Phase 2 - GNINA CNN RESCORING
      Rescores Vina poses with a convolutional neural network, producing a
      CNN affinity (predicted pKd) and a pose confidence score (0-1).

    Phase 3 - RxDock ORTHOGONAL DOCKING
      Docks each candidate de novo using RxDock, an independent docking engine,
      producing an intermolecular energy score.

    Phase 4 - CONSENSUS RANKING
      Ranks candidates by each engine independently, then averages the ranks
      to produce a consensus score. Candidates that rank well across multiple
      engines are more likely to be true binders.

A Flask-based web interface guides you through every step -- no command line
knowledge is needed after initial setup.


REQUIREMENTS
============

  - Windows 10 or Windows 11 (64-bit)
  - Python 3.9 or newer (Python 3.11+ recommended)
  - Internet connection for initial package installation
  - At least 4 GB of free RAM (8 GB recommended for large proteins)
  - About 500 MB of disk space for software and dependencies


STEP-BY-STEP INSTALLATION
==========================

1. INSTALL PYTHON
-----------------

If you do not already have Python installed:

  a) Go to https://www.python.org/downloads/
  b) Download Python 3.11 or newer (click the big yellow button).
  c) Run the installer.

     *** IMPORTANT ***
     On the very first screen of the installer, CHECK THE BOX that says:
         [x] Add Python to PATH
     This is critical. Without it, the commands below will not work.

  d) Click "Install Now" and wait for it to finish.
  e) Verify the installation:
     - Press Windows+R, type "cmd", press Enter
     - In the black Command Prompt window, type:

         python --version

     - You should see something like "Python 3.11.7"
     - If you get an error, restart your computer and try again.


2. INSTALL CONDA (Recommended)
------------------------------

Conda makes it much easier to install RDKit (the chemistry library).
If you already have Anaconda or Miniconda, skip to step (c).

  a) Go to https://conda-forge.org/miniforge/
     Download the Windows x86_64 installer (Miniforge3-Windows-x86_64.exe).

  b) Run the installer. Use default settings. When asked about PATH, select
     "Add Miniforge to my PATH environment variable."

  c) Open "Miniforge Prompt" from the Start Menu (or Command Prompt if you
     added Miniforge to PATH).

  d) Create a dedicated environment:

         conda create -n docking python=3.11 rdkit -y

     This installs Python 3.11 and RDKit together. It may take a few minutes.

  e) Activate the environment:

         conda activate docking

     You should see "(docking)" at the beginning of your prompt.

  f) Install the remaining packages:

         pip install flask meeko

  ALTERNATIVE (pip only, no Conda):
  If you prefer not to use Conda, you can try installing everything with pip:

         pip install rdkit meeko flask

  Note: RDKit installation via pip can sometimes be tricky on Windows.
  The Conda method above is more reliable.


3. DOWNLOAD AUTODOCK VINA
-------------------------

AutoDock Vina is the docking engine that does the actual binding calculations.

  a) Go to https://github.com/ccsb-scripps/AutoDock-Vina/releases
  b) Download the latest Windows binary. It will be named something like:
         vina_1.2.5_win_x86_64.exe
  c) Place the downloaded .exe file directly in the StephenDocking folder
     (the same folder that contains web_app.py).
  d) The app will auto-detect the Vina executable when you run it.
     You do NOT need to rename the file.


4. INSTALL GNINA (Optional - for Hierarchical Mode)
-----------------------------------------------------

GNINA is a deep-learning docking tool that uses convolutional neural networks
to evaluate binding poses. It is required for the Hierarchical screening mode.

  *** IMPORTANT: GNINA has NO native Windows binary. ***
  It is a Linux-only tool. To use GNINA on Windows you must install WSL2
  (Windows Subsystem for Linux), which gives you a Linux environment inside
  Windows. This is officially supported by Microsoft and takes about 10 minutes.

  STEP A: Install WSL2 (one-time setup)
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  Open PowerShell as Administrator and run:

      wsl --install

  This installs Ubuntu by default. Restart your computer when prompted.
  After restart, Ubuntu will open and ask you to create a username and password.

  STEP B: Install GNINA inside WSL
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  Open the Ubuntu terminal (search "Ubuntu" in the Start Menu) and run:

      cd ~
      wget https://github.com/gnina/gnina/releases/download/v1.3.2/gnina.1.3.2
      chmod +x gnina.1.3.2
      sudo mv gnina.1.3.2 /usr/local/bin/gnina

  Verify it works:

      gnina --help

  STEP C: Tell Stephen Docking to use GNINA via WSL
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  In the web UI, set the GNINA executable path to:

      wsl gnina

  The pipeline will automatically call GNINA through WSL. File paths are
  converted between Windows and Linux formats automatically.

  If GNINA is not installed, Hierarchical mode will gracefully skip Phases 2-4
  and produce only the standard Vina results.

  Note: For GPU-accelerated GNINA, your Windows NVIDIA driver must be recent.
  WSL2 shares your GPU automatically -- no separate Linux driver is needed.


5. INSTALL RXDOCK (Optional - for Hierarchical Mode)
-----------------------------------------------------

RxDock is an open-source docking program that provides orthogonal docking
scores for consensus ranking. It is optional even in Hierarchical mode.

  *** IMPORTANT: RxDock has NO native Windows binary. ***
  Like GNINA, it requires WSL2. If you already installed WSL2 in Step 4,
  you can install RxDock in the same Ubuntu environment.

  STEP A: Ensure WSL2 is installed (see Step 4 above)

  STEP B: Install Conda inside WSL (if not already installed)
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  Open the Ubuntu terminal and run:

      wget https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-Linux-x86_64.sh
      bash Miniforge3-Linux-x86_64.sh -b
      ~/miniforge3/bin/conda init
      source ~/.bashrc

  STEP C: Install RxDock via Conda
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
      conda install -c bioconda rxdock -y

  Verify it works:

      rbdock --help
      rbcavity --help

  STEP D: Tell Stephen Docking to use RxDock via WSL
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  In the web UI, set the RxDock executable path to:

      wsl rbdock

  If RxDock is not installed, Phase 3 (orthogonal docking) will be skipped.
  Consensus ranking will use only Vina and GNINA scores.


6. INSTALL P2RANK (Optional - for Auto Pocket Triage)
-------------------------------------------------------

P2Rank is a machine-learning tool that predicts protein-ligand binding sites
from protein structure alone. It is used in the "Auto Pocket Triage" docking
box mode to automatically locate candidate pockets.

  *** IMPORTANT: P2Rank is a Java application and runs on Linux. ***
  On Windows it requires WSL2 (see Step 4 above).

  STEP A: Install Java inside WSL
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  Open the Ubuntu terminal and run:

      sudo apt-get update
      sudo apt-get install -y openjdk-17-jre

  Verify Java is installed:

      java -version

  You should see "openjdk version 17.x.x" or similar.

  STEP B: Download and install P2Rank
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
      cd /opt
      sudo wget https://github.com/rdk/p2rank/releases/download/2.5.1/p2rank_2.5.1.tar.gz
      sudo tar -xzf p2rank_2.5.1.tar.gz
      sudo rm p2rank_2.5.1.tar.gz

  Verify it works:

      /opt/p2rank_2.5.1/prank predict -h

  STEP C: Tell Stephen Docking to use P2Rank via WSL
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  In the web UI (Step 3, Auto Pocket Triage mode), set the P2Rank path to:

      wsl /opt/p2rank_2.5.1/prank


7. INSTALL FPOCKET (Optional - for Auto Pocket Triage)
-------------------------------------------------------

Fpocket is a geometry-based pocket detection tool that measures cavity volumes
using Voronoi tessellation. It is used alongside P2Rank in "Auto Pocket Triage"
mode to validate that predicted pockets have sufficient volume for peptide binding.

  *** IMPORTANT: Fpocket has NO native Windows binary. ***
  It requires WSL2 (see Step 4 above).

  STEP A: Build Fpocket from source inside WSL
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  The conda-forge fpocket package has known issues with PDB parsing.
  Build from source instead (requires git and build-essential):

      sudo apt-get update
      sudo apt-get install -y git build-essential libnetcdf-dev
      cd /tmp
      git clone https://github.com/Discngine/fpocket.git
      cd fpocket
      make
      sudo make install

  Verify it works:

      fpocket -h

  Create a wrapper script so it can be called cleanly from Windows:

      sudo bash -c 'cat > /usr/local/bin/fpocket-wrapper << SCRIPT
#!/bin/bash
exec /usr/local/bin/fpocket "\$@"
SCRIPT'
      sudo chmod +x /usr/local/bin/fpocket-wrapper

  Verify the wrapper works:

      fpocket-wrapper -h

  STEP B: Tell Stephen Docking to use Fpocket via WSL
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  In the web UI (Step 3, Auto Pocket Triage mode), set the Fpocket path to:

      wsl fpocket-wrapper

  AUTO POCKET TRIAGE - HOW IT WORKS
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  When you select "Auto Pocket Triage (P2Rank + Fpocket)" in Step 3:

    1. P2Rank runs ML-based prediction to locate the top 3 binding pockets.
    2. Fpocket runs geometry-based detection to measure cavity volumes.
    3. The pipeline cross-references the two tools by Euclidean distance,
       then selects the highest-ranking P2Rank pocket whose matched Fpocket
       volume exceeds your "Min Pocket Volume" threshold.

  The "Min Pocket Volume" parameter controls which pockets are large enough
  for your peptide. Reference guide:

    Small amino acid (Gly, Ala):              ~60 - 90 Angstroms^3
    Bulky amino acid (Trp, Arg):              ~170 - 230 Angstroms^3
    Unnatural bulky sidechains:               ~250+ Angstroms^3
    Recommended minimum for tetrapeptide anchor: ~300 - 500 Angstroms^3

  Default: 300 Angstroms^3 (suitable for tetrapeptide anchor residues).

  If neither P2Rank nor Fpocket is installed, the "Auto Pocket Triage" mode
  will not produce results. Use Pocket Residues or Manual Coordinates instead.

==========

  1) Open Command Prompt (or Miniforge Prompt if you installed Conda).

  2) If using Conda, activate your environment:

         conda activate docking

  3) Navigate to the StephenDocking folder:

         cd C:\path\to\StephenDocking_v0.7.0

     For example:
         cd C:\Users\YourName\Desktop\StephenDocking_v0.7.0

  4) Start the application:

         python web_app.py

  5) Your web browser should open automatically to:

USING THE WEB INTERFACE
========================

The web interface guides you through 9 steps. Each step is a panel on the page.
Fill them out from top to bottom, then click "Run Pipeline."

- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -

STEP 1: RECEPTOR PDB
---------------------
Upload your protein structure file.

  Supported formats: .pdb, .pdbqt, .cif

  Where to get a PDB file:
    - RCSB Protein Data Bank (https://www.rcsb.org) -- search by protein name
      or PDB ID (e.g., "5H0U"), then download the PDB file.
    - PyMOL -- export your prepared structure as a .pdb file.
    - AlphaFold -- for predicted structures.

  Preparation tips:
    - Remove water molecules (HOH residues) before uploading.
    - Remove co-crystallized ligands unless they define the binding site.
    - Remove duplicate chains if only one copy is needed.
    - The tool will handle protonation at pH 7.3 automatically.

  Click "Choose File", select your PDB, and click Upload. The filename will
  appear below the button to confirm the upload.

- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -

STEP 2: LIGAND INPUT
--------------------
Provide your starting peptide.

  Peptide Sequence:
    Enter the peptide using standard one-letter amino acid codes.
    Examples:
      AGFK      (4-residue peptide: Ala-Gly-Phe-Lys)
      RVKR      (4-residue peptide: Arg-Val-Lys-Arg)
      GHKCU     (Note: non-standard codes may need SMILES instead)

  Alternatively, you can provide a SMILES string for non-standard peptides
  or small molecules.

  Ligand Name:
    Give your peptide a short, descriptive name (e.g., "GHK_tripeptide",
    "P93_lead"). This name will appear in the results table.

- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -

STEP 3: DOCKING BOX
--------------------
Define where on the protein the peptide should bind.

  Option A - Pocket Residues (Recommended):
    Enter key residues at the binding site in the format:
        ALA123, GLY456, HIS789
    The tool will automatically calculate a docking box that covers these
    residues with appropriate padding. This is the easiest and most reliable
    method if you know which residues are in or near the active site.

  Option B - Manual Coordinates:
    Specify the center (X, Y, Z) and size (X, Y, Z) of the docking box
    directly. Use this if you have exact coordinates from another tool
    (e.g., PyMOL or a previous docking study).
    Example: Center 12.5, 34.2, -5.8 / Size 20, 20, 20

  Option C - Default:
    If you leave this blank, the tool will attempt to detect the largest
    pocket on the protein surface automatically using the built-in pocket
    finder. This works well for proteins with a clear, deep binding site
    but may be unreliable for flat surfaces.

- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -

STEP 4: RUN MODE
-----------------
Choose how much optimization to perform.

  Single Dock:
    Docks your peptide once and reports the score. Use this for a quick
    sanity check -- does your peptide bind at all? Takes about 30 seconds.

  Sidechain Only:
    Runs Stage 1 only (sidechain optimization). Tests amino acid
    substitutions at each position. Good for a quick optimization run.
    Takes 10-60 minutes depending on peptide length and settings.

  Full Pipeline (Recommended for production):
    Runs all three stages: Sidechain, Backbone, and Minimize. This gives
    the most thorough optimization. Takes 30 minutes to several hours
    depending on peptide length and exhaustiveness settings.

  Hierarchical (Vina + GNINA + RxDock):
    Runs the full Vina pipeline (all 3 stages), then selects the top 20
    candidates and rescores them with GNINA and RxDock for consensus ranking.
    This provides orthogonal validation across three different scoring engines,
    reducing false positives and increasing confidence in the final ranked list.
    Requires GNINA (mandatory for Phases 2-4) and optionally RxDock (Phase 3).
    When this mode is selected, fields for GNINA and RxDock executable paths
    appear below the run mode selector. See "Installing GNINA and RxDock" below.

- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -

STEP 5: AMINO ACID SELECTION
-----------------------------
Choose which amino acids to test at each position.

  Natural Amino Acids:
    A grid of 20 toggle buttons, one for each natural amino acid.
    Click to select (highlighted) or deselect (gray).

    Group buttons for quick selection:
      [Hydrophobic]  - A, V, L, I, M, F, W, P (nonpolar side chains)
      [Polar]        - S, T, N, Q, Y, C, G     (uncharged polar)
      [Charged]      - D, E, K, R, H           (charged at pH 7)
      [All]          - Select all 20
      [None]         - Deselect all

  Custom Unnatural Amino Acids (UAAs):
    Add non-standard amino acids using SMILES notation. Each UAA SMILES
    must start with [*] which represents the attachment point to the
    peptide backbone (the alpha-carbon).

    Convention:  [*] replaces the alpha-carbon and its hydrogen.
                 Everything after [*] is the side chain.

    Examples:
      [*]CCCC          Norleucine (straight 4-carbon chain)
      [*](C)C          AIB / alpha-aminoisobutyric acid (two methyl groups)
      [*]c1ccccc1      Phenylglycine (phenyl ring directly on alpha-C)
      [*]CCSc1ccccc1   S-benzyl-homocysteine
      [*]CC(=O)N       Beta-asparagine analog

    Enter one SMILES per line in the UAA text box. Give each a short name.

- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -

STEP 6: ADVANCED SETTINGS
--------------------------
Fine-tune the docking parameters. Default values work well for most cases.

  Exhaustiveness (default: 8)
    Controls how thoroughly Vina searches for binding poses. Higher values
    give more reliable results but take longer.
    - 8:  Standard (good for screening)
    - 16: Thorough (good for final results)
    - 32: Very thorough (for publication-quality results)

  Num Modes (default: 9)
    Maximum number of binding poses to generate per docking run.
    The best pose (lowest energy) is used for scoring.

  Max Rounds (default: 3)
    Number of optimization rounds per stage. In each round, the top
    peptides are mutated again. More rounds = deeper optimization but
    longer runtime.

  Top N (default: 5)
    Number of top-scoring peptides to carry forward between rounds.
    Higher values explore more diversity but increase runtime.

  Delta Threshold (default: 0.5 kcal/mol)
    Minimum improvement required to accept a mutation. If a mutation
    improves the score by less than this threshold, it is discarded.
    Lower values keep more marginal improvements.

  Max Residues (default: 5)
    Maximum peptide length for the Minimize stage. Residue deletion
    will not shrink the peptide below this length.

  Poor Binding Cutoff (default: -4.0 kcal/mol)
    Peptides scoring worse (less negative) than this value are flagged
    as poor binders and may be dropped from further optimization.

  Scan C-terminal caps (checkbox, default: off)
    When enabled, the best candidate each round is docked in both the
    deprotonated acid form (COO-) and the amide form (CONH2) at pH 7.3.
    Adds up to 2 extra docks per round per stage.

  N-terminal methylation (checkbox, default: off)
    Replaces all N-H bonds on the N-terminal nitrogen with N-CH3.
    For standard amino acids (primary amine): NH2 -> N(CH3)2 (two methyls).
    For proline (secondary amine): NH -> N-CH3 (one methyl).
    Does NOT modify lysine, arginine, or UAA sidechain amines.
    Adds 1 extra dock per round for the best candidate.

  N-terminal acylation (checkbox, default: off)
    Caps the N-terminal nitrogen with an acyl group, forming an amide bond.
    A number input lets you set the carbon chain length:
      2 = acetyl (CH3-CO-)
      3 = propionyl
      16 = palmitoyl (CH3-(CH2)14-CO-)
    Useful for assessing skin permeability of lipidated peptide hits.
    Adds 1 extra dock per round for the best candidate.

  Custom N-terminal modification (checkbox, default: off)
    Paste any SMILES fragment with [*] marking the bond to the N-terminal
    nitrogen. The [*] atom will be replaced by a direct bond to the N-term.
    Examples:
      [*]C(=O)C           Acetyl group
      [*]C(=O)CCCCC       Hexanoyl group
      [*]C(=O)c1ccccc1    Benzoyl group
    Adds 1 extra dock per round for the best candidate.

  All N-terminal toggles can be active simultaneously (up to 3 extra docks
  per round). They only modify the N-terminal nitrogen and will correctly
  handle proline at the N-terminus.

RUNNING AND VIEWING RESULTS
============================

After clicking "Run Pipeline":

  - A stage indicator shows which stage is currently running
    (Stage 1: Sidechain, Stage 2: Backbone, Stage 3: Minimize).
  - A countdown timer estimates remaining time for the current stage.
  - A live log panel shows real-time output from the docking engine.
  - Between stages, the pipeline pauses at checkpoints so you can review
    intermediate results before continuing to the next stage.

Results Table:
  When complete, a results table appears with the following columns:

    Rank    - Position in the ranking (1 = best binder)
    Name    - Peptide name or identifier
    Score   - Predicted binding energy in kcal/mol (more negative = stronger)
    Stereo  - Stereochemistry annotation (R/S at each chiral center)
    Origin  - Which stage/round produced this variant
    SMILES  - Chemical structure in SMILES notation

  You can:
    - Download the full results as a CSV file for further analysis
    - Download the consensus CSV (hierarchical mode only) with multi-engine
      rankings, CNN pose confidence, and rank variance between engines
    - Download the best complex PDB to visualize in PyMOL or similar
    - Download QC (Quality Control) complex PDBs for specific modification
      categories. These appear only when the corresponding toggle was active:
        * Best D-amino acid complex
        * Best beta-amino acid complex
        * Best unnatural amino acid complex
        * Best C-terminal amide complex
        * Best N-terminal methylated complex
        * Best N-terminal acylated complex
        * Best N-terminal custom modification complex
      Open these in PyMOL or ChimeraX to visually verify that modifications
      are docking in the binding site correctly.


INTERPRETING RESULTS
====================

Binding scores are in kcal/mol. More negative = stronger predicted binding.

  Score Range          Interpretation
  -----------------    ------------------------------------------------
  Above -4.0           Very weak or no meaningful binding
  -4.0 to -6.0        Weak binding (may not be biologically relevant)
  -6.0 to -8.0        Moderate binding (worth investigating)
  -8.0 to -10.0       Strong binding (good lead candidate)
  Below -10.0          Very strong binding (excellent candidate)

Important notes:
  - Always compare scores RELATIVE to your starting peptide. A 2 kcal/mol
    improvement is significant.
  - These are computational predictions. Promising candidates should be
    validated experimentally (SPR, ITC, pull-down assays, etc.).
  - Scores between different proteins are NOT directly comparable.
  - The pipeline tests both R and S stereoisomers at each modified position.
    Check the Stereo column to know which isomer is predicted to bind better.


TROUBLESHOOTING
===============

Problem: "No module named rdkit"
Solution: Install RDKit.
    With Conda:  conda install -c conda-forge rdkit
    With pip:    pip install rdkit

Problem: "No module named meeko"
Solution: pip install meeko

Problem: "No module named flask"
Solution: pip install flask

Problem: "Vina executable not found" or "Vina not found"
Solution: Download AutoDock Vina from the GitHub releases page (see
    Installation Step 3 above) and place the .exe in the StephenDocking
    folder. Then click Auto-Detect or enter the path manually in Step 7.

Problem: "Port 5000 is already in use"
Solution: Another application is using port 5000. Either close that
    application, or run Stephen Docking on a different port:
        python web_app.py --port 5001
    Then go to http://127.0.0.1:5001 in your browser.

Problem: Browser does not open automatically
Solution: Open your browser manually and go to:
    http://127.0.0.1:5000

Problem: "Permission denied" errors
Solution: Right-click Command Prompt and select "Run as Administrator."

Problem: Pipeline hangs or takes very long
Solution: Try reducing Exhaustiveness to 4 and Max Rounds to 2 for a
    faster (but less thorough) run. Also check that your protein is not
    unusually large (>5000 residues).

Problem: "UnicodeDecodeError" or encoding issues
Solution: Make sure your PDB file uses standard ASCII characters. Re-download
    from RCSB if needed.
